# 盯播助手完整生命周期测试方案

## 测试目标

验证从用户在前端配置监控任务到最终收到微信通知的完整数据流是否正常工作。

## 测试环境要求

- Java 后端服务运行在 8080 端口
- Python 爬虫服务运行在 5000 端口
- RabbitMQ 服务正常运行
- 微信小程序开发环境
- 测试用直播房间（建议使用测试房间或已知稳定的直播间）

## 测试用例设计

### 测试用例 1: 开播提醒任务完整流程

#### 前置条件
- 所有服务正常运行
- 用户已登录小程序
- 测试直播间可用

#### 测试步骤

1. **前端操作**
   - 打开小程序，进入任务添加页面
   - 选择平台：哔哩哔哩
   - 输入房间号：测试房间号
   - 选择任务类型：开播提醒
   - 启用微信通知
   - 点击"创建任务"

2. **Java 后端验证**
   - 检查数据库 monitoring_tasks 表是否新增记录
   - 检查 RabbitMQ 是否收到任务消息
   - 验证任务状态为运行中 (status=1)

3. **Python 服务验证**
   - 检查 Python 服务日志是否收到任务消息
   - 验证是否开始监控指定房间
   - 检查 WebSocket 连接是否建立

4. **触发条件测试**
   - 如果房间未开播，手动开启直播
   - 或者使用已开播的房间进行测试

5. **通知验证**
   - 检查 Java 后端是否收到状态变更回调
   - 验证微信订阅消息是否发送成功
   - 确认用户手机端收到通知

#### 预期结果
- 任务成功创建并存入数据库
- RabbitMQ 消息正常传递
- Python 服务开始监控房间
- 直播状态变化时触发通知
- 用户收到微信订阅消息

### 测试用例 2: 关键词匹配任务流程

#### 测试步骤

1. **前端操作**
   - 创建关键词匹配任务
   - 添加关键词："上架"、"新品"、"优惠"
   - 选择商品上架监控类型

2. **后端验证**
   - 检查关键词是否正确保存为 JSON 格式
   - 验证任务消息包含关键词信息

3. **爬虫验证**
   - 检查是否开始监听弹幕
   - 模拟发送包含关键词的弹幕

4. **触发验证**
   - 确认关键词匹配成功
   - 检查通知是否及时发送

### 测试用例 3: Token 刷新机制

#### 测试步骤

1. **模拟 Token 过期**
   - 等待 Access Token 过期（或手动修改过期时间）
   - 发起 API 请求

2. **验证自动刷新**
   - 检查是否自动使用 Refresh Token 获取新 Token
   - 验证原请求是否自动重试成功
   - 确认用户体验无感知

3. **Refresh Token 过期处理**
   - 模拟 Refresh Token 也过期
   - 验证是否跳转到登录页面
   - 检查 Token 是否被正确清除

## 故障排查指南

### 问题 1: 任务创建成功但 Python 服务未收到消息

**可能原因：**
- RabbitMQ 连接问题
- 消息队列配置错误
- Python 服务未正确启动

**排查步骤：**
1. 检查 RabbitMQ 管理界面，确认队列存在且有消息
2. 查看 Python 服务日志，确认是否连接到 RabbitMQ
3. 验证消息格式是否符合预期

### 问题 2: Python 服务收到任务但无法连接直播间

**可能原因：**
- 房间号错误
- 平台 API 限制
- 网络连接问题

**排查步骤：**
1. 手动验证房间号是否正确
2. 检查平台 API 调用日志
3. 确认网络连接正常

### 问题 3: 监控触发但用户未收到通知

**可能原因：**
- 微信订阅消息模板未配置
- 用户未授权订阅消息
- 通知服务配置错误

**排查步骤：**
1. 检查用户是否已授权订阅消息
2. 验证微信模板 ID 配置
3. 查看通知发送日志

### 问题 4: Token 刷新失败

**可能原因：**
- Refresh Token 过期
- Redis 连接问题
- JWT 配置错误

**排查步骤：**
1. 检查 Refresh Token 是否过期
2. 验证 Redis 服务是否正常
3. 查看 JWT 工具类配置
