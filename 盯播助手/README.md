# 盯播助手 (Live Stream Monitoring Assistant)

## 项目简介

盯播助手是一个全栈项目，融合了前端开发、后端高并发处理、数据抓取、大模型AI Agent以及大数据离线分析。该项目旨在为用户提供智能的直播监控服务，能够实时监测主播开播状态、商品上架、抽奖活动等关键事件，并通过AI语义分析理解直播内容。

## 技术栈

### 前端
- **Uni-app** (基于Vue.js) - 一套代码编译微信小程序
- UI组件库: uView

### 后端
- **Java Spring Boot** + MyBatis Plus
- **Redis** - 缓存与高频读取
- **RabbitMQ** - 消息队列，削峰填谷
- **MySQL** - 主数据库

### AI引擎
- **Python** + 大模型API (DeepSeek、Kimi等)
- WebSocket实时数据流处理

### 大数据分析
- **Hadoop生态** (HDFS + MapReduce/Hive) - 离线分析海量弹幕日志

## 核心功能

1. **实时监控** - 监测主播开播状态、商品上架等关键事件
2. **AI语义分析** - 智能理解直播内容，识别抽奖、降价、比赛高潮等状态
3. **智能推送** - 通过微信订阅消息及时通知用户
4. **离线分析** - 分析主播热度趋势，提供智能推荐

## 快速开始

### 环境要求
- JDK 17+
- Node.js 16+
- Python 3.9+
- MySQL 8.0+
- Redis 6.0+
- RabbitMQ 3.8+

### 部署步骤

1. 克隆项目
```bash
git clone <repository-url>
cd 盯播助手
```

2. 配置数据库
```bash
# 执行database目录下的SQL脚本
mysql -u root -p < database/init.sql
```

3. 启动后端服务
```bash
cd stream-backend-java
./mvnw spring-boot:run
```

4. 启动Python服务
```bash
cd stream-spider-python
pip install -r requirements.txt
python main.py
```

5. 启动前端
```bash
cd stream-mini-app
npm install
npm run dev:mp-weixin
```

## 🐛 严重Bug修复清单

### 1. Python Async/Sync 混用问题 🔴
**位置**: `stream-spider-python/main.py:179`
**问题**: Flask同步路由直接调用异步AI服务方法，导致返回coroutine对象而非实际结果
**修复**: 使用`asyncio.run()`包装异步调用或迁移到FastAPI

### 2. Java Spring AOP代理失效 🔴
**位置**: `stream-backend-java/src/main/java/com/streammonitor/service/impl/UserInfoServiceImpl.java:73`
**问题**: 同类内部方法调用绕过Spring AOP代理，导致`@Transactional`注解失效
**修复**: 使用`AopContext.currentProxy()`获取代理对象进行调用

## ⚡ 高并发优化清单

### 3. 注册登录并发安全 (TOCTOU漏洞) 🟡
**位置**: `UserInfoServiceImpl.processUserDatabaseLogic()`
**问题**: 检查-执行模式在高并发下可能导致重复用户创建
**优化**: 使用Redisson分布式锁或数据库唯一索引处理并发冲突

### 4. WebSocket事件循环冲突 🟡
**位置**: `main.py:297`
**问题**: 新线程中运行asyncio WebSocket服务缺少事件循环设置
**优化**: 在新线程中显式创建和设置asyncio事件循环

## 🏗️ 架构优化清单

### 5. 弹幕数据存储优化 🟢
**位置**: `database/init.sql`
**问题**: 全量弹幕数据直接写入MySQL，数据量膨胀影响性能
**优化**: 启用分区表功能，配合定时任务自动清理过期数据

### 6. JWT刷新令牌缺失 🟢
**位置**: `UserController.java`
**问题**: `JwtUtils`已实现刷新方法但缺少对应API接口
**优化**: 添加`/user/refresh-token`接口支持无感Token刷新
